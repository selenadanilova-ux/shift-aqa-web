# Интенсив ШИФТ — Практические задания

Репозиторий для выполнения домашних работ и практических заданий в рамках интенсива ШИФТ.  
Все проекты реализованы на **TypeScript**.

## 🛠️ Стек технологий

* **Язык программирования:** TypeScript
* **Платформа запуска:** Node.js
* **Фреймворк для автотестов:** Playwright
* **Среда разработки (IDE):** Visual Studio Code
* **Система контроля версий:** Git / GitHub

## 🛠️ Настройка окружения

Для запуска проекта локально выполните следующие шаги:

### 1. Предварительные требования
Убедитесь, что на вашем компьютере установлены:
* **Git** (версии 2.x или выше)
* **Node.js** (LTS версия)
* **Visual Studio Code** (с плагинами для TypeScript и автотестов)

### 2. Клонирование репозитория
Склонируйте проект на локальный компьютер:
```bash
git clone https://github.com
cd shift-aqa-web
```

### 3. Сборка и подготовка окружения
 Разверните окружение согласно инструкции из канала «Автотестирование»:
```bash
npm install
```

### 📸 Визуальное тестирование и маскирование (Playwright)
В рамках расширения покрытия реализованы следующие автотесты и шаги:

1. **Актуализация карты страниц**
   * Проверены все адреса из `pages.json`.
   * Автоматически сформированы скриншоты для тех страниц, которые отсутствовали в базовом наборе.

3. **Тестирование интерактивных элементов (FAQ)**

### 🧪 Функциональное тестирование и сквозные сценарии (Playwright)
В рамках расширения тестового покрытия на фреймворке **Playwright** реализованы следующие автоматизированные проверки:

1. **Тестирование комплексной сортировки с пагинацией**
   * **Выбор сортировки**: Реализована активация сортировки товаров по убыванию стоимости.
   * **Сквозной обход**: Написан динамический цикл для автоматического перехода по страницам каталога через кнопки пагинации.
   * **Сбор и валидация данных**: Организован сбор цен со всего массива страниц в единый сквозной массив вместо изолированных проверок каждой страницы по отдельности.
   * **Финальное утверждение**: Добавлено утверждение (assertion), подтверждающее правильный убывающий порядок цен по всему каталогу сайта.

2. **Покрытие тестами обязательного поля ввода «ФИО»**
   * **Позитивные сценарии**: 
     * Проверка граничных значений длины строки (от 1 до 70 символов).
     * Валидация различных раскладок и регистров (кириллица, латиница, UPPERCASE, lowercase).
     * Тестирование допустимых спецсимволов (пробел, дефис).
     * Проверка автоматической разблокировки кнопки отправки после ввода корректных данных и закрытия UI-ошибок.
     * Проверка перехвата успешного ответа бэкенда и появления модального окна об успешной отправке.
   * **Негативные сценарии**:
     * Проверка граничных значений длины строки (0 и 71 символ).
     * Тестирование недопустимых значений (цифры, &, точка, диакритические знаки, пробел).
     * Проверка автоматической разблокировки кнопки отправки после ввода корректных данных и закрытия UI-ошибок.
     * Проверка перехвата успешного ответа бэкенда и появления модального окна об успешной отправке.
     * Тестирование валидации «на лету» в процессе ввода некорректных символов (отображение ошибки и текста-подсказки).
     * Проверка блокировки (`disabled`) кнопки отправки при пустом или невалидном поле.
